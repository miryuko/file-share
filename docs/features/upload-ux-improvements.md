# 上传交互体验优化

## 1. 场景

### 场景 1：拖拽文件上传（两阶段）
- 角色：发送方
- 前置条件：已打开首页，在文件 Tab
- 操作：从文件管理器拖拽 3 个文件到上传区域
- 期望：页面切换为文件审核列表，显示每个文件的类型图标、名称、大小，底部显示总计信息。用户可移除不需要的文件。点击"发送 N 个文件"后开始上传，显示每个文件的进度条、速度和预计剩余时间。上传完成后显示分享码和二维码。

### 场景 2：点击选择文件
- 角色：发送方
- 前置条件：已打开首页，在文件 Tab
- 操作：点击上传区域 → 系统文件选择器弹出 → 选择 1 个文件
- 期望：文件出现在审核列表中，点击"发送 1 个文件"后直接上传（小文件直接上传，大文件分片上传）。

### 场景 3：剪贴板粘贴
- 角色：发送方
- 前置条件：已复制一张图片到剪贴板
- 操作：在发送页面按 Ctrl+V
- 期望：图片文件出现在审核列表中，可像普通文件一样发送。

### 场景 4：取消上传
- 角色：发送方
- 前置条件：正在上传大文件
- 操作：点击"取消上传"按钮
- 期望：正在上传的分片被中止，未开始的文件标记为"已取消"，页面返回选择状态。

### 异常场景
- 网络断开 → 页面顶部显示"网络连接已断开，请检查网络"黄色横幅
- 上传过程中网络出错 → 单个分片失败时自动重试（最多 3 次），重试耗尽后标记该文件为失败，显示"重试"按钮
- 添加文件超过限制 → 弹出警告提示，列出被拒绝的文件及原因
- 全部文件上传失败 → 显示错误信息，提供"发送新文件"按钮
- 部分文件上传失败 → 显示"部分文件上传失败，可以重试"提示，成功的文件仍可通过分享码获取
- 零大小文件 → 提示"文件 xxx 大小为 0，已跳过"

## 2. 功能边界

包含：
- 拖拽/点击/粘贴三种文件添加方式
- 上传前文件审核（查看、移除、再添加）
- 两阶段上传流程（选择审核 → 确认发送）
- 小文件直传（< 5MB），大文件分片上传（≥ 5MB）
- 分片上传自动重试（最多 3 次，指数退避）
- 上传取消
- 上传速度和预计剩余时间显示
- 文件类型图标
- 网络连接状态指示
- 全页拖放覆盖层

不包含：
- 上传暂停/恢复
- 并发上传多个文件
- 上传队列管理
- 文件夹上传
- 过期时间选择器（计划 Phase 4）
- 后台继续上传（页面关闭后恢复）

限制：
- 文件大小/数量/总大小受管理员配置限制（`admin:config`）
- 分片大小固定 5MB
- 直传阈值 5MB
- 重试上限 3 次

## 3. 验收标准
- [ ] 拖拽文件到上传区域，显示文件审核列表
- [ ] 点击上传区域，系统文件选择器弹出，选择后可添加
- [ ] 粘贴剪贴板图片，文件出现在审核列表中
- [ ] 审核列表中可移除单个文件，可添加更多文件
- [ ] 点击"发送"按钮后开始上传，显示进度条
- [ ] 上传过程中显示速度和预计剩余时间
- [ ] 小文件（< 5MB）使用直传，大文件使用分片上传
- [ ] 上传过程中点击"取消"，所有进行中的上传被中止
- [ ] 分片上传失败后自动重试，重试耗尽后标记失败
- [ ] 上传完成后显示分享码和二维码
- [ ] 网络断开时显示连接状态横幅
- [ ] 全页拖放覆盖层在拖拽文件时显示
- [ ] 添加超限文件时显示警告信息
- [ ] 部分上传失败时仍可共享已成功的文件

## 4. API 变更

无新增或修改 API。使用现有端点：

### 直传（新增调用）
- `POST /api/upload/:code/:fileId/direct` — 小文件直传（已有端点，前端之前未使用）
  - Body: `ArrayBuffer`
  - 200: `{ fileId: string; size: number }`

### 已有端点保持不变
- `POST /api/session/create` — 创建上传会话
- `POST /api/upload/:code/:fileId/part?partNumber=N` — 上传分片（新增 AbortSignal 支持）
- `POST /api/upload/:code/:fileId/complete` — 完成上传
- `GET /api/session/:code` — 获取会话信息

## 5. 数据流

### 两阶段上传流程

```
阶段 1：选择 & 审核
  drag / click / paste
    → addFiles(files)
    → 客户端校验（maxFiles, maxFileSize, maxTotalSize, 零大小）
    → 有效文件添加到 selectedFiles[]
    → 返回 AddFilesResult（added, rejected, errors）
    → UI 显示 FileListPreview（类型图标、名称、大小、移除按钮）

阶段 2：上传
  user click "发送"
    → startUpload()
    → createSession(files metadata)
    → for each file:
        ├─ size < 5MB → uploadFileDirect(code, fileId, data, signal)
        └─ size >= 5MB → chunked upload with retry:
            ├─ for each 5MB chunk:
            │   ├─ uploadPart(code, fileId, partNum, data, signal)
            │   ├─ on failure → retry up to 3 times (200/400/800ms backoff)
            │   └─ on abort → mark cancelled, break
            └─ completeUpload(code, fileId, parts, signal)
    → 计算 overallProgress（speed by rolling 3-chunk window, ETA）
    → phase = "completed" | "error" | "cancelled"
```

## 6. 架构对齐

- 存储位置：无新增存储
- 传输路径：纯前端变更，复用现有 R2 存储路径
- 所属 Phase：Phase 4（体验打磨）
- 符合分层死规则：无需后端变更，前端 composable → API → 后端已有的 service 层
- 不影响现有 P2P 传输逻辑

## 7. 降级

| 风险 | 处理 |
|------|------|
| 浏览器不支持 Clipboard API | paste 事件监听静默无操作，拖拽和点击仍可用 |
| 分片上传失败 | 自动重试 3 次，仅对网络/服务器错误（不重试 4xx） |
| 用户取消上传 | AbortController.abort() 中止所有 inflight 请求 |
| 网络断开 | 监听 online/offline 事件，页面顶部横幅提示 |
| 所有文件上传失败 | 显示错误信息，提供"发送新文件"按钮重置 |
| 部分文件上传失败 | 成功的文件通过分享码仍可下载，失败的文件显示重试按钮 |
