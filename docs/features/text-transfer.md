# 剪贴板文本传输 (Phase 4)

## 为什么做

用户场景：手机上复制了一段 URL/密码/笔记，想快速传到电脑上。文件上传太重，文本粘贴 3 秒搞定。不做的后果：每次传简单文本都要走文件上传流程，体验割裂。

## 场景

### 场景 1：发送文本
- 角色：发送方
- 前置条件：已打开首页（发送 tab）
- 操作：在文本框粘贴或输入文本，点击发送
- 期望：显示 6 位分享码，可一键复制

### 场景 2：接收文本
- 角色：接收方  
- 前置条件：持有有效分享码
- 操作：在接收 tab 输入分享码，点击接收
- 期望：展示文本内容，提供一键复制按钮

### 异常场景
- 文本超过 1MB → 提示"文本过长，当前限制 1MB，请使用文件传输"
- 分享码不存在 → 提示"分享码无效，请检查后重试"
- 文本已过期 → 提示"内容已过期，文本保留 1 小时后自动清理"
- 服务不可用 → 提示"服务异常，请稍后重试"

## 功能边界

包含：粘贴/输入文本、生成 6 位分享码、接收方查看并一键复制
不包含：Markdown 渲染、代码语法高亮、富文本编辑、文本修改后重新分享
限制：单条文本 ≤ 1MB

## 验收标准
- [ ] 粘贴一段 URL，点击发送，生成 6 位码
- [ ] 接收方输入码，文本完整展示，无截断
- [ ] 一键复制按钮点击后内容写入系统剪贴板
- [ ] 粘贴 2MB 内容，提示"文本过长"
- [ ] 过期后扫码，提示"内容已过期"
- [ ] 输入不存在的分享码，提示"分享码无效"

## 架构对齐
- 存储位置：KV，键前缀 `text:`
- 传输路径：纯服务端中转，不涉及 P2P 和 R2
- 与现有功能的关系：复用 Cron cleanup 机制（扫描 `text:` 前缀）
- 所属 Phase：Phase 4（体验打磨）

## API 变更

### POST /api/text/create
- 认证：匿名（按 IP 限流）
- Body: `{ content: string }`
- 201: `{ code: string, expiresAt: number }`
- 400: 内容为空或超长（>1MB）
- 429: 同一 IP 超过频率限制

### GET /api/text/{code}
- 认证：匿名
- 200: `{ content: string, expiresAt: number, createdAt: number }`
- 404: 分享码不存在
- 410: 内容已过期

## 数据模型

KV 新增键空间 `text:`：
```
text:{code} → {
  content: string,
  createdAt: number,
  expiresAt: number
}
```
